# Seed corpus для AFL++ фаззинга libtiff

Данная директория содержит минимальный набор входных TIFF-файлов для запуска фаззинга.

## Файлы

| Файл | Формат | Размер | Описание |
|------|--------|--------|----------|
| `minimal_1x1_gray.tif` | LE TIFF | 111 bytes | Минимальный little-endian TIFF, 1×1 px, 8-bit grayscale |
| `bigendian_4x4_rgb.tif` | BE TIFF | 248 bytes | Big-endian TIFF (Motorola), 4×4 px, RGB |

## Структура TIFF

TIFF-файл состоит из:
1. **Заголовок (8 байт)**: байт-порядок (`II`=LE / `MM`=BE), magic=42, смещение IFD
2. **IFD (Image File Directory)**: набор тегов с метаданными изображения
3. **Данные пикселей**: сжатые или несжатые

## Почему именно эти файлы

AFL++ мутирует входные данные, отслеживая покрытие кода. Небольшие валидные TIFF-файлы позволяют:
- быстро пройти парсинг заголовка и попасть в "глубокий" код обработки,
- охватить оба byte-order варианта (LE и BE),
- обеспечить разные пути кода для разных форматов пикселей.
